## 氣泡排序法 (Bubble Sort)

* **原理簡述：**透過重複掃描要排序的數列，一次比較兩個相鄰的元素，若順序錯誤（如前大後小）則將其互換。每一輪比較都會將當前未排序區間的最大值，像「氣泡」一樣逐一推擠到數列的最右端，直到掃描全過程都沒有發生任何交換，即代表排序完成。
* **複雜度分析：**
  * 最佳時間複雜度：O(n)  
  * 平均時間複雜度：O(n^2)
  * 最壞時間複雜度：O(n^2) 
  * 空間複雜度：O(1)
  * 結論：氣泡排序屬於空間複雜度為O(1)的原地排序演算法。雖然在數列已排序的最佳情況下能以 O(n)的時間提早結束，但其平均與最壞時間複雜度皆為O(n^2)。由於需要頻繁進行相鄰元素的比較與交換，在處理大量隨機數據時效能表現較差，實務上通常僅用於演算法邏輯的教學示範。

## 選擇排序法 (Selection Sort)

* **原理簡述：**將數列分為「已排序」與「未排序」兩個區間。演算法每一輪都會完整掃描未排序區間，精準找出其中的「最小值」，將該最小值與未排序區間的第一個元素進行交換，將其納入已排序區間。反覆執行此動作，直到所有元素皆排序完畢。有效減少了元素交換的次數。
* **複雜度分析：**
  * 最佳時間複雜度：O(n^2) 
  * 平均時間複雜度：O(n^2) 
  * 最壞時間複雜度：O(n^2) 
  * 空間複雜度：O(1)
  * 結論：選擇排序同樣具備O(1)的極低空間複雜度。然而，無論數據的初始分佈是整齊還是混亂，它都必須完整走訪剩餘的未排序區間以尋找極值。因此，其最佳、平均與最壞時間複雜度皆固定落在 O(n^2)，是一種效能極度受限且無法因數據「部分排序」而獲益的演算法。

## 插入排序法 (Insertion Sort)

* **原理簡述：**運作方式類似於玩家在手中整理撲克牌。將陣列分為「已排序」與「未排序」區間，每次從未排序區間抽取出第一個元素，並在已排序區間中由後往前逐一比對，尋找合適的空位並將其「插入」。
* **複雜度分析：**
  * 最佳時間複雜度：O(n) 
  * 平均時間複雜度：O(n^2) 
  * 最壞時間複雜度：O(n^2) 
  * 空間複雜度：O(1) 
  * 結論：作為空間複雜度O(1)的原地排序，插入排序的最大優勢在於處理「近乎排序完成」或「小規模」的數列時，時間複雜度可優化至O(n)。儘管其平均與最壞時間複雜度仍為O(n^2)，但在實務上，它經常被現代高階排序演算法當成處理極小區塊數據的底層輔助演算法。

## 合併排序法 (Merge Sort)

* **原理簡述：**採用分治法。首先將整個陣列不斷對半遞迴拆分，直到每個子陣列只剩下單一元素（單一元素視為已排序狀態）。接著，將這些相鄰的小陣列進行兩兩合併，在合併的過程中依序比較大小並寫入暫存陣列中，最終像拉鍊一樣組合回一個完整且穩定的有序陣列。
* **複雜度分析：**
  * 最佳時間複雜度：O(n log n) 
  * 平均時間複雜度：O(n log n) 
  * 最壞時間複雜度：O(n log n) 
  * 空間複雜度：O(n) 
  * 結論：合併排序是犧牲空間複雜度（需額外配置O(n)的暫存陣列空間）來換取極致時間效能的典型代表。基於分治法嚴謹的「切分與合併」結構，其最佳、平均與最壞時間複雜度皆穩定維持在 O(n \log n)。這種特性，使其特別適合處理對穩定性要求高、且記憶體資源充足的大型資料集。

## 快速排序法 (Quick Sort)

* **原理簡述：**同樣使用分治法的策略。演算法會先從數列中挑選出一個元素作為基準值。接著進行分區操作：將所有小於基準值的元素移到其左側，大於基準值的元素移到右側。分區完成後，基準值便會落在最終的正確位置。最後，對基準值左右兩側的子陣列反覆執行相同步驟，直到整體排序完成。
* **複雜度分析：**
  * 最佳時間複雜度：O(n log n) 
  * 平均時間複雜度：O(n log n)
  * 最壞時間複雜度：O(n^2)
  * 空間複雜度：O(log n) 
  * 結論：快速排序在一般隨機數據下，平均與最佳時間複雜度皆為 O(n \log n)，且因內部操作簡單、對快取記憶體友善，是實務上執行速度最快的演算法之一。儘管在極端數據與選錯基準值的最壞情況下會退化至O(n^2)，且遞迴呼叫需消耗平均O(log n) 的空間複雜度，但透過優化基準值選取策略，它依然是多數程式語言標準函式庫的預設首選。
